Persistence.

Transitive does not provide an ORM, or even the concept of a "Domain Model."

Transitive deals with the shuffling of datastructures around to interested parties. Hydrating domain objects with that data and writing that data to disk are currently out of scope.

Modern applications use many different kinds of "databases" for different kinds of data -- RDBMS, Document Stores, Redis, HBase, and even using RESTful APIs to other services. Each of these has different transactional semantics and unique features. Transitive is mostly ambivelant about how you persist your data. Instead of providing a least-common-denominator abstraction, Transitive just reqiures that your perisistence layer emits events when things change.

You can use persistence layers that don't have this ability, but then you'll have to implement it in the application layer, which is pretty easy. [Guide: Implementing a Broadcasting Data store]()

Domain Models are tricky.
